# Wordrobe 衣橱
衣橱管理 iOS 应用，目标是提供现代、简洁、可扩展的衣物管理体验，包括图片上传、分类标签、智能搭配建议与展示。

主要实现功能（建议优先级按 MVP → 增强功能 列出）

- **用户档案与设置（MVP）**: 支持基本用户偏好设置（性别、尺码、常用风格），用于改善搭配建议。
- **拍照与图片导入（MVP）**: 从相机或相册添加衣物图片，支持简单的裁剪与旋转。
- **衣物条目管理（MVP）**: 为每件衣物创建条目，包含：照片、名称、类别（上衣/下装/外套/鞋子/配饰）、颜色、品牌、尺码、材质、备注与自定义标签。
- **分类与筛选（MVP）**: 按类别、颜色、标签、季节、风格、是否已穿等维度筛选或分组浏览。
- **收藏与展示（MVP）**: 收藏常用/喜爱搭配，网格和列表两种简洁展示模式。
- **搭配方案生成（增强）**: 根据类别与颜色规则，自动生成若干可穿搭组合；允许用户保存、评分与编辑搭配。
- **日程与穿搭历史（增强）**: 记录穿搭历史并支持在日历中查看或为未来日期安排穿搭。
- **导入/导出与备份（增强）**: 支持 iCloud 备份/同步，或导出/导入 JSON/ZIP 打包数据和图片。
- **搜索与智能标签（增强）**: 基于图像识别或关键词自动建议标签（可选整合 Core ML 模型）。
- **分享与社交（可选）**: 将搭配或单品以图片/链接分享至社交平台。

UI / 交互指导

- **视觉风格**: 极简扁平化设计，使用留白、清晰的卡片和高质量图片展示，主色建议使用柔和的中性色配合一到两个强调色。
- **主要导航**: 使用底部 `TabBar`，包含：`衣橱`（网格）、`搭配`（建议与已保存搭配）、`日历`（历史/计划）、`设置`。
- **单品详情**: 单页卡片式详情，图片顶部可左右滑动切换图片，底部显示属性、标签与操作按钮（编辑/删除/收藏/添加到搭配）。
- **操作快捷**: 长按卡片显示常用操作，支持多选批量标签/删除/导出。

技术与实现建议

- **平台与语言**: iOS 17+，使用 Swift + SwiftUI，并优先考虑 `SwiftData` 作为持久层。
- **数据存储**: 使用 `SwiftData` 作为本地模型层，结合 `CloudKit` 做 iCloud 同步（在 SwiftData 基础上实现的同步或通过 CloudKit API 将数据与图片上云）。
- **图片处理**: 使用 `Photos` 框架 + `SwiftUI` 的 `PhotosPicker`，将图片存为文件并在模型中保存文件名/相对路径；同步时使用 `CKAsset` 上传图片二进制，客户端保留缓存与缩略图（可用 `SDWebImageSwiftUI`）。
- **同步与备份**: 优先使用 `CloudKit` 实现 iCloud 数据与文件同步；实现离线支持、变更监听与冲突解决（字段级合并或最后修改优先）。保留 JSON 导出/导入用于手动迁移与备份。
- **智能功能**: 使用 `Core ML` + `Vision` 做颜色检测/类别建议；对 SwiftData 的模型可扩展属性用于记录识别元数据。
- **测试**: 使用 `Swift Testing` / UI Tests 覆盖关键流程，并增加离线/同步/冲突场景的集成测试。

MVP 验收标准（可用于里程碑验收）

- 能从相机或相册添加衣物图片并保存为条目（含类别与至少一个标签）。
- 能在 `衣橱` 页面以网格查看已添加的衣物，支持按类别/标签筛选。
- 能查看单件衣物的详情并进行编辑与删除操作。
- 有一个简单的 `搭配` 页面，能基于已有单品生成至少 3 套可浏览的静态搭配（基于规则即可）。
- 基本设置页面能保存用户偏好，且应用在搭配建议中起作用。

项目与开发建议

- 采用模块化设计：`Model`（Core Data/Realm）、`Services`（图片、同步、备份）、`UI`（各页面和组件）、`AI/Rules`（搭配引擎）。
- 从最小可行版本（MVP）开始迭代：先实现图片导入、单品管理、筛选、搭配规则，再逐步加入云同步与智能标签。
